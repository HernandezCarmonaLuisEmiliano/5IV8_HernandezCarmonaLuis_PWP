Todos los formularios en la web, manejan metodos para el envio de diferentes peticiones que realiza el usuario. Considereando la arquitectura Cliente-Servidor, el manejo de de las peticiones se hace por medio del protocolo http y actualmente por https, cada peticion debe ser atendida en un tiempo de respuesta conforme al modelo TCP/IP. Los metodos principales de los formularios son:

    • Get:

      este metodo se encarga de enviar la informacion del formulario por medio de la salida standard (conocida como url), del navegador, esto propicia que puedan existir inyecciones de codigo realizando violaciones a la seguridad del sistema

      Post:

      este metodo se encarga de enviar la informacion para poder registrar un cambio o un nuevo registro dentro del sistema enviandola igual por el metodo estandar, mediante un ocultamiento, esto significa que no es visible para el usuario lo que nos ofrece un mayor grado de seguridad en el manejo de la informacion


    • En resumen los metodos son:

    • Get: Sirve para enviar y realizar operaciones de lectura o consulta
      Post: Sirve para enviar informacion de un recurso al servidor con el fin de registrarlo
      Delete: Sirve para eliminar un recurso especifico dentro del servidor, obteniendo una respuesta de actualizacion
      Put: Sirve para almacenar la informacion de un recurso particular y conectarse al servicio para realizar la actualizacion
      Head: Sirve para regresar el codigo de respuesta de cualquier cabecera HTTP asociada a un recurso
